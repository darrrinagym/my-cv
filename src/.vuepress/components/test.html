 Frameworks
In the early 2010s, I discovered Bootstrap prototyping. Unfortunately, I will not say the exact date. But it was often the best way to avoid double work. Especially when back-end administrative interfaces are being created and you need to think about usability rather than design differences. There are many frameworks out there now, and I usually choose the one that best suits the task. Once you have a library of custom components ready, you can create interfaces from your own components as a framework. Which also speeds up the development process.

 ## ... and nocode When I use nocode

When to write code, and when to use NOCODE tools - it depends on the task, deadline, budget.

For MVP testing, it is often faster to use no-code tools. Yes, this limits usability and does not usually scale. But if we need to sit on something right now, let's first make a stool, and then design an ergonomic chair.
